# 口播知识短视频生产工作流（Agent 通用版）

> 本文件是给任何 AI Agent（Kimi / Codex / Claude Code / Cursor 等）使用的完整工作流说明。
> Agent 读完后应能独立执行任意一条视频从选题到发布的全流程。
> 不依赖特定工具：每一步只定义「输入 / 动作 / 输出 / 确认门」，工具可替换（括号内为当前在用实现，可换等价物）。

---

## 0. 总览

```
选题 ──→ 口播稿 ──→ 拍摄+初剪（用户完成）──→ 精剪管线 ──→ 封面 ──→ 发布
         ↑______________ 复盘数据回流（播放/完播/评论）_______________↑
```

**四道硬性确认门（Agent 不得跳过）：**

| 门 | 确认内容 | 未确认前禁止 |
|---|---|---|
| G1 选题门 | 选题 + 唯一承诺一句话 | 不写稿 |
| G2 稿件门 | 口播稿逐字确认（稿件=权威文本） | 不动工 |
| G3 抽检门 | 10 秒抽检样片（带完整声音方案） | 不渲染全片 |
| G4 发布门 | 完整审片版通看 + 封面文字逐字确认 | 不出交付版/封面图 |

**核心原则：**
1. 用户手工初剪的视频 = 唯一锁定母版，剪辑结构不再改。
2. 口播最终稿 = 权威文本；语音转录只用于时间对齐，名词校正逐条经用户确认。
3. 一切修改先出小样（关键帧静帧 / 10 秒抽检），用户确认后才扩展全片。
4. 抽检样片必须带完整声音方案（BGM + 响度收口），让用户在抽检阶段听到的就是最终效果。
5. 版本化命名，永不覆盖旧稿。

---

## 1. 阶段一：选题

**输入：** 热点 / 用户想法 / 历史视频复盘数据
**输出：** 选题卡（一句话唯一承诺 + 目标人群 + 证据素材清单）
**模板：** `选题/选题卡-模板.md`（复盘数据从 `复盘/发布记录.md` 取）

**步骤：**
1. 候选选题来源：AI 工具圈近期热点（新模型/新功能发布、限免、榜单变化）、用户实操中踩过的坑、评论区高频问题。
2. 每个候选写成「选题卡」：
   - 唯一承诺：这条视频看完观众能拿走什么（一句话，必须可被正文兑现）。
   - 目标人群：零基础 / 进阶 / 同行。
   - 证据素材：需要哪些真实录屏、截图、数据（选题时就盘点，避免写到一半发现没素材）。
   - 反差点/钩子：为什么现在看、为什么是你讲。
3. 多个候选时给出推荐排序和理由，用户拍板（**G1**）。

**当前实践参考：** 已验证有效的选题模式 = 热点工具 + 立即可用的实操路径（模式示例：「别学XX了，直接用」「不会XX，一个按钮搞定」「从工具A迁移到工具B」）。

---

## 2. 阶段二：口播稿

**输入：** 确认的选题卡
**输出：** 逐字口播稿（权威文本）

**步骤：**
1. 结构模板（已验证，约 3–5 分钟 / 230–260 字每分钟）：
   - 开场钩子（≤15 秒）：抛出反差或利益点。
   - 正文：步骤化演示（「第一步…第二步…」），每个论点配一个真实证据（录屏/截图/数据）。
   - 收尾：行动号召 + 品牌收尾（「我是<频道名>，下期视频再见」——触发片尾品牌卡）。
2. 写作纪律：
   - 口语化短句，允许自然临场发挥空间；不写书面长句。
   - 专有名词定稿（产品名/版本号必须写准，这是后期字幕校正的基准）。
   - 每个「最」「第一」「NO.1」等表述必须有可核验的证据画面。
3. 用户逐字确认（**G2**）。确认后此稿成为全流程权威文本，后续字幕与其逐字对齐校验。

**之后由用户完成：** 拍摄 + 粗剪（去废片、调语速），产出锁定母版放入项目目录。

## 3. 阶段三：精剪管线（Agent 主战场）

**输入：** 锁定母版（用户初剪视频）+ 权威口播稿
**输出：** 高码率发布交付版 MP4

### 3.0 工程初始化

- **统一走脚手架**：`scripts/init_video.sh 第N条 /path/to/用户初剪母版.mov` —— 自动建目录、硬链接母版并记录 inode、ffprobe 实测生成 master.json、输出字幕/覆盖层骨架 JSON。全程勾选清单见 `references/checklist.md`。
- 每条视频一个独立工作目录（如 `edit/<第N条>-master/`），下分 `transcripts/`、`assets/`、`hyperframes-keyframes/`、`review-sample-*/`、`review-v1/`。
- 确认规格：分辨率 / 帧率 / 时长（以 ffprobe 实测为准），默认 1920×1080，帧率沿用母版。
- **素材核验**：所有用户给的截图/录屏素材，先 ffprobe + 目检确认实际内容与文件名/口播一致（曾有「单榜第一」素材实际是别的榜单的教训）；不一致则联网找真实素材替换并经用户确认。

### 3.1 转录 + 字幕时间线

1. 词级转录锁定母版（当前：ElevenLabs Scribe；备用：whisper），独立缓存 JSON。
2. 专有名词校正：列出全部疑似错词（如 CalCode→Claude Code），逐条经用户确认后替换。
3. 生成字幕时间线，固定规则：
   - **字幕不使用标点符号**（，。、；：？！""''（）《》…— 全部删除；数字小数点如 2.5 保留；英文词组间标点删除时按需补空格）。
   - 断句按语义自然切分；以权威稿件做 canon 校验（拼接字幕去标点 == 最终稿逐字一致）。
   - 样式：`Noto Sans JP` 700 字重、暖白正文、82% 深色圆角底板；行动词/数字暖黄强调、风险词红色；单行 62px / 双行 57px；底部留出字幕安全区（约 154px）。
   - 若用户中途改锚点/延时：字幕平移必须处理「跨界条」（横跨插入点的字幕按实测 token 拆分），不能只按起点归类。
4. 先出 3 张静帧样式审片（普通单行 / 重点强调 / 双行长句）确认样式，再全量。

### 3.2 覆盖层（HyperFrames 程序化图形）

1. 按口播内容规划覆盖层槽位时间线（通常 8–15 段），两类：
   - **真实素材段**：录屏（画框适配：如 1248×1080 居中 + 两侧深色侧板 + 2px 暖黄分隔线）、截图（长截图预裁切保证行不被切断）。
   - **程序化图形段**：步骤卡、比例条、工具阵列、行动卡等。
2. 设计系统（已固化，新视频沿用）：
   - 覆盖层整体包 `.fg` 90% 不透明度，字幕保持 100% 最顶层。
   - 90% 透明叠加下蓝红强调色会对比度不足：蓝用 `#6dafff`、红用 `#ef8a85`、amber 用 `#f4c161`（均已检查器实测 ≥4.5:1）。
   - 场景动效：旧场景交接前约 0.4s 缓出（power2/sine），新场景 0.58–0.92s 分层缓入（power3/sine）。
   - **防闪屏**：所有 `.reveal/.hero/.topline` CSS 初始 opacity 0，GSAP fromTo 接管；`.fg` 退出淡化后补 tl.set 硬清除。
   - 开场 0–3s 可无覆盖层纯母版开场；片尾有品牌口播时加品牌收尾卡（定格，保留实拍动态背景）。
3. 先出全场景代表关键帧（静帧，1920×1080 PNG）给用户审，确认前不渲染视频（**确认门 G3 前置**）。
4. 工具类特效增加发布风险检查：
   - 不把大量 Agent、工具、应用名称集中做成名称墙、Logo阵列、应用商店式卡片或推荐下载清单。
   - 不在装饰性特效中加入下载按钮、安装提示、二维码、外链入口或类似导流元素。
   - 只保留当前口播和证据所必需的少量工具名称；能用功能示意、输入输出对比或抽象图形表达时，不堆产品名。
   - 关键帧审查和最终成片复核时，单独判断画面是否可能被理解为引导下载、批量推广或商业推荐。
   - 以上是保守风险控制，不冒充任何平台已经确认的具体审核规则。

### 3.3 BGM 入库与审核

- 音乐库 `edit/bgm-library/`（首次使用按 setup.md 第四步入库一条已确认 BGM，命名例：`bgm_003-明亮轻快-已确认.wav`）。
- 选曲约束（用户确立）：禁小调压迫、暗色氛围垫、电影式悬疑、渐强、重低频；明亮、松弛、友好、全程稳定。
- 换新 BGM 时先发纯音乐文件给用户单独审核，通过后才混入样片。

### 3.4 10 秒抽检样片（G3，固定规则）

- 选窗口原则：覆盖一个场景交接点 + 一个高信息密度段；必要时自主前移以便自验交接。
- **必须带完整声音方案**（用户确立的固定规则）：
  1. 人声：loudnorm 两遍法（第一遍测量、第二遍 measured_* 注入），目标 I=-16 / TP=-3（含长静音段的窗口用 I=-13.2）。
  2. BGM：-15.4dB 左右垫底（约 -30 LUFS）+ 头 0.3s / 尾 0.5s 淡化。
  3. amix normalize=0 → +2.2dB → alimiter（短片 0.80 起步，长片 0.79）。
  4. 成片目标 **-14±0.3 LUFS / ≤-1.5 dBTP**，以 ebur128 实测为准；视频流 copy 不重编码。
- 输出 10 秒样片，用户通看确认后才进全片。
- 以上混音链已脚本化为 `scripts/mix_finalize.py`（一条命令自动完成两遍法+BGM垫底+限幅梯度+实测验收，stdout 只出一行 JSON 结论）。

### 3.5 完整审片版

1. 同一合成规则扩展全片；**响度参数必须按全片实测重算**（全片响度与抽检段不同，禁止照抄）。
2. 渲染超限（约 300s / 单次）时分段：分段点选「场景间纯母版空档 + 字幕空档 + 帧对齐」三重条件位置，ffmpeg -c copy 无损拼接。
3. BGM 循环：acrossfade 0.5s，循环缝精确落在素材首尾相接处。
4. 验证清单（每次必做）：
   - 全解码零报错；实测响度/真峰值达标。
   - 抽 8–10 代表帧目检：字幕最顶层、无标点、90% 透明覆盖层、无闪屏、交接干净。
   - 音字同步证据链：RMS 包络互相关（lag≈0）+ whisper DTW token 偏差表（每条 ≤0.15s）。
   - 渲染工程资产 inode 核验（换母版后必须强制重链 + samefile 断言，防陈旧硬链接）。

### 3.6 高码率发布交付版

- 用户通看确认后，同一工程 `-q high`（16 Mbps 目标）重渲染 + 同一混音链，视频流 copy。
- 封装：H.264 High Profile / BT.709 / yuv420p / AAC 48kHz 双声道 256k / faststart；必须用 `ffprobe` 回读声道数。
- 记录 SHA-256；与审片版同刻抽帧比对（像素差 ≤2/255 为纯编码噪声）。
- 命名：`第N条-发布交付版-1080pXX-高码率.mp4`。

## 4. 阶段四：封面 + 发布

**输入：** 交付版成片 + 字幕文本
**输出：** 3:4 封面 PNG + 发布标题/文案

**固定流程（封面 STYLE_GUIDE 摘要）：**
1. 从字幕提炼正文能兑现的**唯一承诺**。
2. 分开给出「发布标题」（完整表达）和「封面大字」（2–3 行、每行 4–7 字、移动端第一眼识别）——两者不必逐字相同，但承诺必须一致。
3. **用户逐字确认文字后才出图**（G4）。
4. 选人物帧：全片每 2s 抽帧 + 拉普拉斯方差锐度排名 + 安静窗口细扫；优先正对镜头、口型闭合、手势推动标题含义的帧；保留人物身份特征（标志性配饰/发型/服装）。
5. AI 生成 2:3 底图（蓝黑科技背景 + 发光代码/数据元素 + 大字：第一行蓝青/橙、第二行白、结论行最大号黄白、粗重窄体深色描边；禁止额外出文字/Logo/水印/序号标注）。
6. PIL 后期：上下裁切（约 85/86 px）为 3:4（顺便去除「AI生成」小字）→ 缩放 1086×1448 → 左上角合成品牌角标（v3 细描边版，宽 330px、留边 28px；**绝不让生成模型画角标**）。
7. 核对清单：文字逐字正确、人物/手指无畸形、身份特征无漂移、无第三方水印、270px 缩略图大字可读。
8. 版本化存 `edit/cover/output/第N条视频封面-vX.png`。
9. 发布前自查（必做）：基于最终成片检查标题兑现、字幕、素材权利、隐私、AI与营销声明，以及工具名称墙、下载/安装暗示和批量推广风险；外部合规skill只能作为辅助，不能替代逐帧检查。
10. 发布后：记录发布时间、标题、封面版本，为复盘留档。

## 5. 复盘回流（建议新增）

- 记录表：`复盘/发布记录.md`（每片一行：日期/标题/封面版本/成片路径/T+3 数据/一条经验）。
- 发布后 T+3 天记录：播放、完播、点赞/评论/收藏、涨粉。
- 对照选题卡的「唯一承诺」判断兑现度，沉淀到下轮选题。
- 固定复盘问题：钩子前 15 秒留存如何？哪个论点评论区反响最大？封面 CTR 如何？

## 6. 经验沉淀库（历次踩坑，Agent 必读）

| # | 教训 | 规则 |
|---|---|---|
| 1 | 素材文件名与实际内容不符 | 素材先 ffprobe + 目检再进时间线 |
| 2 | 换母版后渲染用了旧文件 | 资产强制重链 + inode/samefile 断言 |
| 3 | 时间线插入段后字幕错位 | 跨界字幕按 token 实测拆分，不按起点归类 |
| 4 | 场景进入闪屏 | 元素 CSS 初始 opacity 0 + GSAP fromTo；退出后硬清除 |
| 5 | 长静音段 alimiter 硬拉增益损音质 | 改人声 loudnorm 目标，不用 alimiter 硬拉 |
| 6 | 抽检段响度 ≠ 全片响度 | 全片参数永远按全片实测重算 |
| 7 | 90% 透明叠加对比度不足 | 强调色提亮，以检查器实测 ≥4.5:1 为准 |
| 8 | whisper 窗口 token 有 ±0.2s 噪声 | 多测量源交叉（全片 DTW + 包络互相关），单次超界 ≠ 真超界 |
| 9 | 长片/短片限幅器不同 | 短片 alimiter 0.80 起步；>150s 用 0.79 |
| 10 | 响度微调无需重跑混音链 | 纯 volume 电平调整线性收口（-0.4dB → -0.4 LU） |

## 7. Skill 路由表（哪个环节调哪个 skill / 跑哪个脚本）

| 环节 | 用什么 | 说明 |
|---|---|---|
| Agent 粗剪模式（多素材转录/选段/粗剪） | 执行环境的粗剪/转录类 skill（如 video-use） | 仅在用户不交初剪、要 Agent 从原始素材剪起时启用；用户自剪模式跳过。工作区 takes/EDL/转录缓存规范可沿用该 skill 的约定 |
| 精剪管线（字幕/覆盖层/混音/审片） | 本 skill 自带脚本 | `init_video.sh` / `review_gen/generate.py` / `mix_finalize.py`；多条规则继承自粗剪工作法（词级转录缓存、30ms 淡化、分段+无损拼接、字幕最后合成、词边界切点） |
| 选题打分 / 发布后 T+3 复盘 | `cheat-on-content` 家族（cheat-score / cheat-status 等） | 可选；与 `选题/选题卡` 和 `复盘/发布记录.md` 配合 |
| 素材联网核实 | `agent-reach` / `web-access` | 素材货不对板时找真实素材 |
| 覆盖层程序化图形 | `hyperframes` 家族（hyperframes / hyperframes-cli / hyperframes-keyframes） | 关键帧、check 校验、渲染 |
| 封面底图 | `image_generation`（Kimi 插件）/ Codex 自带生图 skill | 按执行环境选：在 Kimi 里用 image_generation 插件，在 Codex 里用 Codex 自带生图 skill；只画底图，角标一律 PIL 后期合成 |
| 发布前自查 | `traffic-compass` | 违禁词/限流风险 + 播放区间预测 |

原则：创作判断类环节挂 skill；工程执行类环节跑自建脚本（避免每次重读 skill 说明烧 token）。

## 8. 一页速查（Agent 每次开工先读这里）

```
开工 → 确认当前阶段卡在哪道门（G1选题/G2稿/G3抽检/G4发布）
剪辑顺序：核验母版规格 → 转录+名词确认 → 字幕时间线(去标点) →
         覆盖层槽位 → 关键帧静帧审 → BGM确认 → 10s抽检(带完整混音) →
         全片审片版(响度重算) → 高码率交付版 → 封面(先文字后图)
硬性数字：-14±0.3 LUFS / ≤-1.5 dBTP / 字幕无标点 / 覆盖层90% / 
         Noto Sans JP 700 / 蓝#6dafff 红#ef8a85 amber#f4c161 /
         封面 1086×1448 / 角标左上330px留边28px
工具入口：混音收口 = scripts/mix_finalize.py（一条命令）
         审片合成到交付 = scripts/review_gen/generate.py --check --render --mix
         （需要已确认的关键帧工程和三份时间线 JSON，不负责自动创作场景）
         （三份 JSON 驱动：master/subtitles/overlay，场景钩子按类型分发）
永不：跳过确认门 / 覆盖旧版本 / 让AI画角标 / 照抄抽检响度参数
```
